# Head 메타데이터 {#head-metadata}

## Head 메타데이터 커스터마이징 {#customizing-head-metadata}

Docusaurus는 자동으로 `<html>`, `<head>`, `<body>`에 유용한 페이지 메타데이터를 설정합니다. Markdown 파일에서 `<head>` 태그를 사용하여 추가 메타데이터를 추가하거나 기존 메타데이터를 재정의할 수 있습니다:

```md title="markdown-features-head-metadata.mdx"
---
id: head-metadata
title: Head Metadata
---

<!-- highlight-start -->
<head>
  <html className="some-extra-html-class" />
  <body className="other-extra-body-class" />
  <title>Head Metadata customized title!</title>
  <meta charSet="utf-8" />
  <meta name="twitter:card" content="summary" />
  <link rel="canonical" href="https://docusaurus.io/docs/markdown-features/head-metadata" />
</head>
<!-- highlight-end -->

# Head Metadata

My text
```

```mdx-code-block
<head>
  <html className="some-extra-html-class" />
  <body className="other-extra-body-class" />
  <title>Head Metadata customized title!</title>
  <meta charSet="utf-8" />
  <meta name="twitter:card" content="summary" />
  <link rel="canonical" href="https://docusaurus.io/docs/markdown-features/head-metadata" />
</head>
```

이 `<head>` 선언은 데모로 현재 Markdown 문서에 추가되었습니다. 브라우저 개발자 도구를 열어 이 페이지의 메타데이터가 어떻게 변경되었는지 확인해보세요.

:::note

이 기능은 Docusaurus [`<Head>`](./../../docusaurus-core.mdx#head) 컴포넌트를 기반으로 구축되었습니다. 자세한 문서는 [react-helmet](https://github.com/nfl/react-helmet)을 참조하세요.

:::

:::tip 일반적인 SEO에는 이것이 필요하지 않습니다

콘텐츠 플러그인(예: docs와 blog)은 `description`, `keywords`, `image`와 같은 프론트 매터 옵션을 제공하며, 이는 자동으로 `description`과 `og:description` 모두에 적용됩니다. 반면 `<head>` 태그를 사용할 때는 두 메타데이터 태그를 수동으로 선언해야 합니다.

:::

## Markdown 페이지 설명 {#markdown-page-description}

Markdown 페이지의 설명 메타데이터는 head 메타데이터 외에도 여러 곳에서 사용될 수 있습니다. 예를 들어, docs 플러그인의 [생성된 카테고리 인덱스](../docs/sidebar/items.mdx#generated-index-page)는 문서 카드에 설명 메타데이터를 사용합니다.

기본적으로 설명은 첫 번째로 의미 있는 줄의 내용을 기준으로 하며, 해당 내용을 일반 텍스트로 변환하려는 처리가 포함됩니다. 예를 들어, 다음 파일은...

```md
# Title

Main content... May contain some [links](./file.mdx) or **emphasis**.
```

...기본 설명으로 "Main content... May contain some links or emphasis"를 가지게 됩니다. 하지만, **이 방식이 항상 완벽하게 작동하도록 설계된 것은 아닙니다**. 합리적인 설명을 생성하지 못하는 경우, 적절한 설명이 생성되지 않을 경우, 프론트 매터를 통해 직접 설명을 지정할 수 있습니다:

```md
---
description: 이 설명은 기본값을 덮어씁니다.
---

# Title

Main content... May contain some [links](./file.mdx) or **emphasis**.
```
