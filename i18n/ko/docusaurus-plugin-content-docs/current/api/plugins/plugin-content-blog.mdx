# 📦 plugin-content-blog {#plugin-content-blog}

import APITable from '@site/src/components/APITable';

[블로그](blog.mdx) 기능을 제공하며 Docusaurus의 기본 블로그 플러그인입니다.

:::warning 일부 기능은 프로덕션에서만 사용 가능합니다

[피드 기능](../../blog.mdx#feed)은 빌드 출력을 추출하여 작동하며, **프로덕션에서만 활성화**됩니다.

:::

## 설치 {#installation}

```bash npm2yarn
npm install --save @docusaurus/plugin-content-blog
```

:::tip

`@docusaurus/preset-classic` 프리셋을 사용하는 경우, 이 플러그인을 의존성으로 설치할 필요가 없습니다.

[프리셋 옵션](../../using-plugins.mdx#docusauruspreset-classic)을 통해 이 플러그인을 구성할 수 있습니다.

:::

#### `FeedXSLTOptions` {#FeedXSLTOptions}

브라우저가 [XSLT](https://developer.mozilla.org/en-US/docs/Web/XSLT)를 사용하여 블로그 XML 피드를 깔끔하게 렌더링할 수 있도록 합니다.

- 블로그가 내장된 `.xsl` 및 `.css` 파일을 사용하여 블로그 피드의 스타일을 지정하도록 하려면 `true`를 사용하세요
- 기능을 비활성화하려면 falsy 값(`undefined | null | false`)을 사용하세요
- 블로그 콘텐츠 폴더를 기준으로 사용자 정의 `.xsl` 파일의 경로를 제공하려면 `string`을 사용하세요. 관례상 정확히 같은 이름의 `.css` 파일도 제공해야 합니다.

```ts
type FeedXSLTOptions =
  | boolean
  | undefined
  | null
  | {
      rss?: string | boolean | null | undefined;
      atom?: string | boolean | null | undefined;
    };
```

### 예제 구성 {#ex-config}

프리셋 옵션이나 플러그인 옵션을 통해 이 플러그인을 구성할 수 있습니다.

:::tip

대부분의 Docusaurus 사용자는 프리셋 옵션을 통해 이 플러그인을 구성합니다.

:::

```js config-tabs
// 프리셋 옵션: blog
// 플러그인 옵션: @docusaurus/plugin-content-blog

const config = {
  path: 'blog',
  // 간단한 사용 사례: 문자열 editUrl
  // editUrl: 'https://github.com/facebook/docusaurus/edit/main/website/',
  // 고급 사용 사례: 함수형 editUrl
  editUrl: ({locale, blogDirPath, blogPath, permalink}) =>
    `https://github.com/facebook/docusaurus/edit/main/website/${blogDirPath}/${blogPath}`,
  editLocalizedFiles: false,
  blogTitle: '블로그 제목',
  blogDescription: '블로그',
  blogSidebarCount: 5,
  blogSidebarTitle: '모든 게시물',
  routeBasePath: 'blog',
  include: ['**/*.{md,mdx}'],
  exclude: [
    '**/_*.{js,jsx,ts,tsx,md,mdx}',
    '**/_*/**',
    '**/*.test.{js,jsx,ts,tsx}',
    '**/__tests__/**',
  ],
  postsPerPage: 10,
  blogListComponent: '@theme/BlogListPage',
  blogPostComponent: '@theme/BlogPostPage',
  blogTagsListComponent: '@theme/BlogTagsListPage',
  blogTagsPostsComponent: '@theme/BlogTagsPostsPage',
  remarkPlugins: [require('./my-remark-plugin')],
  rehypePlugins: [],
  beforeDefaultRemarkPlugins: [],
  beforeDefaultRehypePlugins: [],
  truncateMarker: /<!--\s*(truncate)\s*-->/,
  showReadingTime: true,
  feedOptions: {
    type: '',
    title: '',
    description: '',
    copyright: '',
    language: undefined,
    createFeedItems: async (params) => {
      const {blogPosts, defaultCreateFeedItems, ...rest} = params;
      return defaultCreateFeedItems({
        // 피드에는 가장 최근의 블로그 포스트 10개만 유지
        blogPosts: blogPosts.filter((item, index) => index < 10),
        ...rest,
      });
    },
  },
};
```

## 작성자 파일 {#authors-file}

[`authors` 플러그인 옵션](#authors)을 사용하여 YAML 작성자 파일의 경로를 구성하세요.

통상적으로, 플러그인은 블로그 콘텐츠 폴더의 루트에서 `authors.yml` 파일을 찾습니다.

이 파일은 [전역 블로그 작성자](../../blog.mdx#global-authors)의 목록을 포함할 수 있습니다. [`authors` 프런트 매터](#markdown-front-matter)를 통해 Markdown 파일에서 이러한 작성자를 키로 참조할 수 있습니다.

### 타입 {#authors-file-types}

제공된 작성자 파일의 YAML 내용은 다음과 같은 형태를 따라야 합니다:

```tsx
type AuthorsMapInput = {
  [authorKey: string]: AuthorInput;
};

type AuthorInput = {
  name?: string;
  title?: string;
  description?: string;
  imageURL?: string;
  url?: string;
  email?: string;
  page?: boolean | {permalink: string};
  socials?: Record<string, string>;
  [customAuthorAttribute: string]: unknown;
};
```

### 예시 {#authors-file-example}

```yml title="tags.yml"
slorber:
  name: Sébastien Lorber
  title: Docusaurus 관리자
  url: https://sebastienlorber.com
  image_url: https://github.com/slorber.png
  page: true
  socials:
    x: sebastienlorber
    github: slorber

jmarcey:
  name: Joel Marcey
  title: Docusaurus 1의 공동 제작자
  url: https://github.com/JoelMarcey
  image_url: https://github.com/JoelMarcey.png
  email: jimarcey@gmail.com
  page:
    permalink: '/joel-marcey'
  socials:
    x: joelmarcey
    github: JoelMarcey
```

```md title="blog/my-blog-post.md"

authors: [slorber, jmarcey]
```

## i18n {#i18n}

먼저 [i18n 소개](../../i18n/i18n-introduction.mdx)를 읽어보세요.

### 번역 파일 위치 {#translation-files-location}

- **기본 경로**: `website/i18n/[locale]/docusaurus-plugin-content-blog`
- **다중 인스턴스 경로**: `website/i18n/[locale]/docusaurus-plugin-content-blog-[pluginId]`
- **JSON 파일**: [`docusaurus write-translations`](../../cli.mdx#docusaurus-write-translations-sitedir)로 추출됨
- **Markdown 파일**: `website/i18n/[locale]/docusaurus-plugin-content-blog`

### 예시 파일 시스템 구조 {#example-file-system-structure}

```bash
website/i18n/[locale]/docusaurus-plugin-content-blog
│
│ # website/blog의 번역
├── authors.yml
├── first-blog-post.md
├── second-blog-post.md
│
│ # 렌더링될 플러그인 옵션에 대한 번역
└── options.json
```
