# 📦 plugin-content-docs {#plugin-content-docs}

import APITable from '@site/src/components/APITable';

Docusaurus의 기본 문서 플러그인으로, [문서](../../guides/docs/docs-introduction.mdx) 기능을 제공합니다.

## 설치 {#installation}

```bash npm2yarn
npm install --save @docusaurus/plugin-content-docs
```

:::tip

`@docusaurus/preset-classic` 프리셋을 사용하는 경우, 이 플러그인을 의존성으로 설치할 필요가 없습니다.

[프리셋 옵션](../../using-plugins.mdx#docusauruspreset-classic)을 통해 이 플러그인을 구성할 수 있습니다.

:::

### 타입 {#types}

#### `EditUrlFunction` {#EditUrlFunction}

```ts
type EditUrlFunction = (params: {
  version: string;
  versionDocsDirPath: string;
  docPath: string;
  permalink: string;
  locale: string;
}) => string | undefined;
```

#### `PrefixParser` {#PrefixParser}

```ts
type PrefixParser = (filename: string) => {
  filename: string;
  numberPrefix?: number;
};
```

#### `SidebarGenerator` {#SidebarGenerator}

```ts
type SidebarGenerator = (generatorArgs: {
  /** 변환할 "autogenerated" 유형의 사이드바 항목입니다. */
  item: {type: 'autogenerated'; dirName: string};
  /** 이 사이드바가 속한 버전에 대한 유용한 메타데이터입니다. */
  version: {contentPath: string; versionName: string};
  /** 해당 버전의 모든 문서(필터링되지 않음)입니다. */
  docs: {
    id: string;
    title: string;
    frontMatter: DocFrontMatter & Record<string, unknown>;
    source: string;
    sourceDirName: string;
    sidebarPosition?: number | undefined;
  }[];
  /** 이 플러그인에 대해 구성된 숫자 접두사 파서입니다. */
  numberPrefixParser: PrefixParser;
  /** 재정의할 수 있는 기본 카테고리 인덱스 매처입니다. */
  isCategoryIndex: CategoryIndexMatcher;
  /**
   * key는 문서 콘텐츠 디렉토리를 기준으로 한 경로이고,
   * value는 카테고리 메타데이터 파일의 내용입니다.
   */
  categoriesMetadata: {[filePath: string]: CategoryMetadata};
  /**
   * Docusaurus의 기본 사이드바 생성 로직을 재사용/개선하는 데
   * 유용합니다.
   */
  defaultSidebarItemsGenerator: SidebarGenerator;
  // 사이드바 항목 배열을 반환합니다 — sidebars.js에서 선언할 수 있는 것과
  // 동일하지만 약식은 제외됩니다. https://docusaurus.io/docs/sidebar/items 참조
}) => Promise<SidebarItem[]>;

type CategoryIndexMatcher = (param: {
  /** 확장자가 없는 파일 이름 */
  fileName: string;
  /**
   * 가장 낮은 수준에서 가장 높은 수준까지의 디렉토리 목록입니다.
   * 디렉토리 이름이 없으면 directories는 ['.']입니다.
   */
  directories: string[];
  /** 앞에 점이 있는 확장자 */
  extension: string;
}) => boolean;
```

#### `VersionsConfig` {#VersionsConfig}

```ts
type VersionConfig = {
  /**
   * 버전의 기본 경로로, `baseUrl` + `routeBasePath`에
   * 추가됩니다.
   */
  path?: string;
  /** 배지, 드롭다운 등에서 사용할 버전의 레이블입니다. */
  label?: string;
  /** 해당 버전의 문서 상단에 표시할 배너입니다. */
  banner?: 'none' | 'unreleased' | 'unmaintained';
  /** 각 문서 상단에 버전 레이블이 있는 배지를 표시합니다. */
  badge?: boolean;
  /** 검색 엔진이 이 버전을 색인화하지 못하게 합니다. */
  noIndex?: boolean;
  /** 각 문서의 <html> 요소에 사용자 정의 클래스 이름을 추가합니다. */
  className?: string;
};

type VersionsConfig = {[versionName: string]: VersionConfig};
```

### 예제 구성 {#ex-config}

프리셋 옵션이나 플러그인 옵션을 통해 이 플러그인을 구성할 수 있습니다.

:::tip

대부분의 Docusaurus 사용자는 프리셋 옵션을 통해 이 플러그인을 구성합니다.

:::

```js config-tabs
// 프리셋 옵션: docs
// 플러그인 옵션: @docusaurus/plugin-content-docs

const config = {
  path: 'docs',
  breadcrumbs: true,
  // 간단한 사용 사례: 문자열 editUrl
  // editUrl: 'https://github.com/facebook/docusaurus/edit/main/website/',
  // 고급 사용 사례: 함수형 editUrl
  editUrl: ({versionDocsDirPath, docPath}) =>
    `https://github.com/facebook/docusaurus/edit/main/website/${versionDocsDirPath}/${docPath}`,
  editLocalizedFiles: false,
  editCurrentVersion: false,
  routeBasePath: 'docs',
  include: ['**/*.md', '**/*.mdx'],
  exclude: [
    '**/_*.{js,jsx,ts,tsx,md,mdx}',
    '**/_*/**',
    '**/*.test.{js,jsx,ts,tsx}',
    '**/__tests__/**',
  ],
  sidebarPath: 'sidebars.js',
  async sidebarItemsGenerator({
    defaultSidebarItemsGenerator,
    numberPrefixParser,
    item,
    version,
    docs,
    isCategoryIndex,
  }) {
    // 제공된 데이터를 사용하여 사용자 정의 사이드바 슬라이스 생성
    return [
      {type: 'doc', id: 'intro'},
      {
        type: 'category',
        label: 'Tutorials',
        items: [
          {type: 'doc', id: 'tutorial1'},
          {type: 'doc', id: 'tutorial2'},
        ],
      },
    ];
  },
  numberPrefixParser(filename) {
    // 잠재적인 숫자 접두사를 추출하기 위한 자체 로직 구현
    const numberPrefix = findNumberPrefix(filename);
    // 접두사를 찾음: 정리된 파일 이름과 함께 반환
    if (numberPrefix) {
      return {
        numberPrefix,
        filename: filename.replace(prefix, ''),
      };
    }
    // 숫자 접두사를 찾지 못함
    return {numberPrefix: undefined, filename};
  },
  docsRootComponent: '@theme/DocsRoot',
  docVersionRootComponent: '@theme/DocVersionRoot',
  docRootComponent: '@theme/DocRoot',
  docItemComponent: '@theme/DocItem',
  remarkPlugins: [require('./my-remark-plugin')],
  rehypePlugins: [],
  beforeDefaultRemarkPlugins: [],
  beforeDefaultRehypePlugins: [],
  showLastUpdateAuthor: false,
  showLastUpdateTime: false,
  disableVersioning: false,
  includeCurrentVersion: true,
  lastVersion: undefined,
  versions: {
    current: {
      label: 'Android SDK v2.0.0 (WIP)',
      path: 'android-2.0.0',
      banner: 'none',
    },
    '1.0.0': {
      label: 'Android SDK v1.0.0',
      path: 'android-1.0.0',
      banner: 'unmaintained',
    },
  },
  onlyIncludeVersions: ['current', '1.0.0', '2.0.0'],
};
```

## Markdown 프런트 매터 {#markdown-front-matter}

Markdown 문서는 양쪽에 `---` 줄로 둘러싸인 다음과 같은 Markdown [프런트 매터](../../guides/markdown-features/markdown-features-intro.mdx#front-matter) 메타데이터 필드를 사용할 수 있습니다.

허용되는 필드:

```mdx-code-block
<APITable>
```

| 이름 | 타입 | 기본값 | 설명 |
| --- | --- | --- | --- |
| `id` | `string` | 파일 경로 (폴더 포함, 확장자 제외) | 고유한 문서 ID입니다. |
| `title` | `string` | Markdown 제목 또는 `id` | 문서의 텍스트 제목입니다. 페이지 메타데이터에 사용되며 여러 곳(사이드바, 다음/이전 버튼 등)에서 대체값으로 사용됩니다. Markdown 제목이 없는 경우 문서 상단에 자동으로 추가됩니다. |
| `title_meta` | `string` | `frontMatter.title` | `<head>`의 `<title>`과 `og:title`에 사용되는 문서의 SEO 메타데이터 제목입니다. 표시되는 제목과 SEO 제목이 다를 때 `frontMatter.title`을 재정의할 수 있습니다. |
| `pagination_label` | `string` | `sidebar_label` 또는 `title` | 이 문서의 문서 다음/이전 버튼에 사용되는 텍스트입니다. |
| `sidebar_label` | `string` | `title` | 이 문서에 대해 문서 사이드바에 표시되는 텍스트입니다. |
| `sidebar_position` | `number` | 기본 순서 | `autogenerated` 사이드바 항목을 사용할 때 생성된 사이드바 슬라이스 내의 문서 위치를 제어합니다. [자동 생성된 사이드바 메타데이터](/docs/sidebar/autogenerated#autogenerated-sidebar-metadata)도 참조하세요. |
| `sidebar_class_name` | `string` | `undefined` | 자동 생성된 사이드바를 사용할 때 해당 사이드바 레이블에 특별한 클래스 이름을 부여합니다. |
| `sidebar_custom_props` | `object` | `undefined` | 이 문서를 참조하는 사이드바 항목에 [사용자 정의 속성](../../guides/docs/sidebar/index.mdx#passing-custom-props)을 할당합니다. |
