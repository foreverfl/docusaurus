# 📦 create-docusaurus

기능적인 Docusaurus 앱을 즉시 설정할 수 있도록 도와주는 유틸리티입니다.

## 사용법 {#usage}

```bash
npx create-docusaurus@latest [name] [template] [rootDir]
```

`name` 인자는 사이트의 경로와 생성된 앱의 package.json의 `name` 필드로 사용됩니다. 절대 경로이거나 `rootDir`에 대한 상대 경로일 수 있습니다.

`template` 인자는 다음 중 하나일 수 있습니다:

- `classic`: 클래식 템플릿 사용 (권장)
- `facebook`: Facebook/Meta 템플릿 사용, Meta 관련 설정이 포함되어 있음
- Git 저장소 URL (시작이 `https://` 또는 `git@`인), 대상 위치에 클론될 수 있음
- CWD에 대한 상대적인 로컬 파일 경로, 대상 위치에 복사될 파일들을 포함

`rootDir`은 사이트 디렉토리의 절대 경로를 해석하는 데 사용됩니다. 기본값은 CWD입니다.

:::warning

이 명령어는 모든 기능을 사용할 수 있도록 대화형 셸에서 사용하는 것이 좋습니다.

:::

## 옵션 {#options}

### `-t, --typescript` {#typescript}

템플릿 인자가 인식된 이름일 때 사용됩니다. 현재는 `classic`만이 TypeScript 변형을 제공합니다.

### `-g, --git-strategy` {#git-strategy}

템플릿 인자가 git 저장소일 때 사용됩니다. 다음 중 하나여야 합니다:

- `deep`: 전체 git 히스토리 보존
- `shallow`: `--depth=1`로 클론
- `copy`: shallow 클론을 수행하지만 git 저장소를 생성하지 않음
- `custom`: 사용자 정의 git 클론 명령어 입력. 프롬프트가 표시됩니다. `git clone --depth 10`과 같이 작성할 수 있으며, 저장소 URL과 대상 디렉토리가 자동으로 추가됩니다.

### `-p, --package-manager` {#package-manager}

값은 `npm`, `yarn`, `pnpm`, 또는 `bun` 중 하나여야 합니다. 명시적으로 제공되지 않은 경우, Docusaurus는 다음을 기반으로 추론합니다:

- CWD에 이미 존재하는 잠금 파일 (예: 기존 프로젝트에서 웹사이트를 설정하는 경우)
- `create-docusaurus`를 호출하는 데 사용된 명령어 (예: `npm init`, `npx`, `yarn create`, `bunx` 등)
- 모든 추론이 불가능한 경우, 대화형 프롬프트가 사용됨

### `-s, --skip-install` {#skip-install}

제공된 경우, Docusaurus는 앱 생성 후 자동으로 종속성을 설치하지 않습니다. `--package-manager` 옵션은 실제로 종속성을 설치할 때만 유용합니다.
