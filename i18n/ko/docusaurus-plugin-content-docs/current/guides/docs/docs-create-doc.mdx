# 문서 생성하기

`greeting.md` Markdown 파일을 생성하여 `docs` 디렉토리 아래에 위치시키세요.

```bash
website # 사이트의 루트 디렉토리
├── docs
│   └── greeting.md
├── src
│   └── pages
├── docusaurus.config.js
├── ...
```

```md
---
description: 풍부한 콘텐츠가 포함된 문서 페이지를 생성합니다.
---

# Docusaurus에서 인사드립니다

오픈 소스 프로젝트를 위한 문서 사이트를 만들 준비가 되셨나요?

## 헤더

오른쪽 상단의 목차에 표시됩니다

이를 통해 사용자는 스크롤을 내리거나 많이 읽지 않고도 이 페이지가 무엇에 관한 것인지 알 수 있습니다.

## 기본적으로 h2와 h3만 목차에 포함됩니다.

문서별로 또는 테마 구성에서 목차 제목 수준을 구성할 수 있습니다.

헤더는 계층 구조가 명확하도록 잘 간격이 맞춰져 있습니다.

- 목록은 도움이 됩니다
- 사용자가 기억해야 할
- 핵심 포인트를 제시하는 데
  - 그리고 중첩할 수 있습니다
    - 여러 번
```

:::note

`docs` 디렉토리 아래의 밑줄(`_`)로 시작하는 모든 파일은 "부분" 페이지로 취급되어 기본적으로 무시됩니다.

[부분 페이지 가져오기](../markdown-features/markdown-features-react.mdx#importing-markdown)에 대해 자세히 알아보세요.

:::

## 문서 프론트매터 {#doc-front-matter}

[프론트매터](../markdown-features/markdown-features-intro.mdx#front-matter)는 문서 페이지에 추가 메타데이터를 제공하는 데 사용됩니다. 프론트매터는 선택사항입니다. Docusaurus는 프론트매터 없이도 필요한 모든 메타데이터를 추론할 수 있습니다. 예를 들어, 아래에 소개된 [문서 태그](#doc-tags) 기능은 프론트매터 사용이 필요합니다. 가능한 모든 필드는 [API 문서](../../api/plugins/plugin-content-docs.mdx#markdown-front-matter)를 참조하세요.

## 문서 태그 {#doc-tags}

선택적으로 문서 페이지에 태그를 추가할 수 있습니다. 이는 [문서 사이드바](./sidebar/index.mdx) 외에 또 다른 분류 방식으로 활용됩니다. 태그는 프론트매터에 레이블 목록으로 전달됩니다:

```md title="docs/my-doc.md"
---
id: doc-with-tags
title: A doc with tags
tags:
  - Demo
  - Getting started
---
```

:::tip

태그는 `tags: [Demo, Getting started]`와 같이 선언할 수도 있습니다.

가능한 모든 [Yaml 배열 구문](https://www.w3schools.io/file/yaml-arrays/)에 대해 자세히 알아보세요.

:::

## 폴더 구조 구성하기 {#organizing-folder-structure}

`docs` 폴더 아래의 Markdown 파일 배치는 Docusaurus 콘텐츠 생성에 여러 영향을 미칠 수 있습니다. 하지만 대부분은 파일 구조와 분리될 수 있습니다.

### 문서 ID {#document-id}

모든 문서는 고유한 `id`를 가집니다. 기본적으로 문서 `id`는 루트 docs 디렉토리를 기준으로 한 문서의 이름(확장자 제외)입니다.

예를 들어, `greeting.md`의 ID는 `greeting`이고, `guide/hello.md`의 ID는 `guide/hello`입니다.

```bash
website # 사이트의 루트 디렉토리
└── docs
   ├── greeting.md
   └── guide
      └── hello.md
```

하지만 `id`의 **마지막 부분**은 프론트매터에서 사용자가 정의할 수 있습니다. 예를 들어, `guide/hello.md`의 내용이 아래와 같이 정의되면 최종 `id`는 `guide/part1`입니다.

```md
---
id: part1
---

Lorem ipsum
```

ID는 사이드바를 수동으로 작성하거나 문서 관련 레이아웃 컴포넌트나 훅을 사용할 때 문서를 참조하는 데 사용됩니다.

### 문서 URL {#doc-urls}

기본적으로 문서의 URL 위치는 `docs` 폴더를 기준으로 한 파일 경로이며, 몇 가지 예외가 있습니다. 즉, 파일 이름이 다음 중 하나인 경우 파일 이름이 URL에 포함되지 않습니다:

- `index`로 명명된 경우(대소문자 구분 없음): `docs/Guides/index.md`
- `README`로 명명된 경우(대소문자 구분 없음): `docs/Guides/README.mdx`
- 상위 폴더와 같은 이름인 경우: `docs/Guides/Guides.md`

모든 경우에, 기본 슬러그는 `/index`, `/README`, 또는 중복된 `/Guides` 세그먼트 없이 `/Guides`만 됩니다.

:::note

이 규칙은 [카테고리 인덱스 규칙](./sidebar/autogenerated.mdx#category-index-convention)과 정확히 동일합니다. 하지만 `isCategoryIndex` 구성은 문서 URL에 영향을 _주지 않습니다_.

:::

`slug` 프론트매터를 사용하여 문서의 URL을 변경하세요.

예를 들어, 사이트 구조가 다음과 같다고 가정해 봅시다:

```bash
website # 사이트의 루트 디렉토리
└── docs
    └── guide
        └── hello.md
```

기본적으로 `hello.md`는 `/docs/guide/hello`에서 사용할 수 있습니다. URL 위치를 `/docs/bonjour`로 변경할 수 있습니다:

```md
---
slug: /bonjour
---

Lorem ipsum
```

`slug`는 기본적으로 `/docs`인 문서 플러그인의 `routeBasePath`에 추가됩니다. URL에서 `/docs` 부분을 제거하는 방법은 [Docs 전용 모드](docs-introduction.mdx#docs-only-mode)를 참조하세요.

:::note

다음을 사용할 수 있습니다:

- 절대 슬러그: `slug: /mySlug`, `slug: /`...
- 상대 슬러그: `slug: mySlug`, `slug: ./../mySlug`...

:::

문서를 루트에서 사용할 수 있게 하고 `https://docusaurus.io/docs/`와 같은 경로를 가지려면 슬러그 프론트매터를 사용할 수 있습니다:

```md
---
id: my-home-doc
slug: /
---

Lorem ipsum
```

### 사이드바 {#sidebars}

[자동 생성된 사이드바](./sidebar/autogenerated.mdx)를 사용할 때 파일 구조가 사이드바 구조를 결정합니다.

파일 시스템 구성에 대한 우리의 권장사항은 다음과 같습니다. `sidebars.js` 파일을 수동으로 작성할 필요가 없도록, 파일 시스템이 사이드바 구조를 반영하도록 만들고, `slug` 프론트매터를 사용하여 각 문서의 URL을 커스터마이징하세요.
