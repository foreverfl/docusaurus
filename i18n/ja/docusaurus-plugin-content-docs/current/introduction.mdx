# 概要

⚡️ Docusaurusは、**美しいドキュメントサイトを瞬時に作成**するお手伝いをします。

💸 カスタムな技術スタックの構築にはコストがかかります。代わりに、**コンテンツに集中**してMarkdownファイルを書くだけにしましょう。

💥 さらに多くの機能が必要ですか？バージョン管理、国際化（i18n）、検索、テーマのカスタマイズなどの**高度な機能**を使ってみましょう。

💅 インスピレーションを得るために、[**最高のDocusaurusサイト**](/showcase?tags=favorite)をチェックしましょう。

🧐 Docusaurusは**静的サイトジェネレーター**です。高速なクライアントサイドナビゲーション機能を備えた**シングルページアプリケーション**を構築し、**React**のすべての機能を活用してサイトをインタラクティブにします。基本的に**ドキュメント機能**を提供しますが、**あらゆる種類のサイト**（個人ウェブサイト、製品、ブログ、マーケティングランディングページなど）を作成するのに使用できます。

![](/img/slash-introducing.svg)

## クイックスタート ⏱️ {#fast-track}

遊びながら**5分**でDocusaurusを理解しましょう。

新しいDocusaurusサイトを作成し、**とても短い**組み込みチュートリアルに従ってください。

[Node.js](https://nodejs.org/en/download/)をインストールし、新しいDocusaurusサイトを作成します：

```bash
npx create-docusaurus@latest my-website classic
```

サイトを起動します：

```bash
cd my-website
npx docusaurus start
```

[`http://localhost:3000`](http://localhost:3000)を開いてチュートリアルに従ってください。

:::tip

[**docusaurus.new**](https://docusaurus.new)を使用して、ブラウザで直接Docusaurusをテストしてみましょう！

または、オンラインで[**5分チュートリアル**](https://tutorial.docusaurus.io)を読んでみましょう。

:::

## v1からの移行 {#migrating-from-v1}

Docusaurus v2+は、完全に現代化されたツールチェーンを活用してDocusaurus v1を完全に書き直したバージョンです。[v2の公式リリース](https://docusaurus.io/blog/2022/08/01/announcing-docusaurus-2.0)以降、私たちはDocusaurus v1の代わりに**Docusaurus v2+を使用**することを強く推奨しています。Docusaurus v1は非推奨となっています。

すでに多くの[ユーザー](/showcase)がDocusaurus v2+を使用しています（[トレンド](https://www.npmtrends.com/docusaurus-vs-@docusaurus/core)）。

**以下の場合にDocusaurus v2+を使用してください：**

- :white_check_mark: 最新のJamstackドキュメントサイトが必要な場合
- :white_check_mark: クライアントサイドルーティングを備えたシングルページアプリケーション（SPA）が必要な場合
- :white_check_mark: ReactとMDXのすべての機能を活用したい場合
- :white_check_mark: IE11のサポートが必要ない場合

**以下の場合に[Docusaurus v1](https://v1.docusaurus.io/)を使用してください：**

- :x: シングルページアプリケーション（SPA）を望まない場合
- :x: IE11のサポートが必要な場合（...本当に必要ですか？IEは[すでに寿命が終わり](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11)、もはや公式にサポートされていません）

v2+にアップグレードしたい既存のv1ユーザーは、[移行ガイド](./migration/index.mdx)を参照できます。

## 機能 {#features}

Docusaurusは、開発者と貢献者の体験に高い注意を払って構築されました。

- ⚛️ **💚とReactで作られています**:

  - Reactで拡張およびカスタマイズ
  - 独自のReactコンポーネントを提供してサイトのブラウジング体験を完全に制御

- 🔌 **プラグイン可能**:

  - 基本テンプレートでサイトをブートストラップした後、高度な機能とプラグインを使用
  - プラグインをオープンソースとして公開してコミュニティと共有

- ✂️ **開発者体験**:
  - 今すぐドキュメント作成を開始
  - 貢献者がより簡単に維持できるよう、汎用的な設定エントリーポイントを提供
  - 変更に対する稲妻のように速い増分ビルドによるホットリロード
  - パスベースのコードとデータの分割
  - GitHub Pages、Netlify、Vercelおよびその他のデプロイメントサービスに簡単に公開

私たちの共通の目標は、ユーザーが必要なものを素早く見つけ、製品をよりよく理解できるようにすることです。ドキュメントサイトを正しく、よく構築できるように、私たちのベストプラクティスを共有します。

- 🎯 **SEOフレンドリー**:

  - 可能なすべてのパスに対してHTMLファイルが静的に生成されます。
  - ページごとのSEOにより、ユーザーが直面している問題に直接関連する公式ドキュメントにすぐにたどり着けるようにします。

- 📝 **MDXサポート**:

  - MarkdownにJSXとReactを含むインタラクティブなコンポーネントを書けます。
  - ライブエディターでコードを共有して、ユーザーが製品をすぐに気に入るようにします。

- 🔍 **検索**：サイト全体を検索できます。

- 💾 **ドキュメントバージョン管理**：プロジェクトのリリースとドキュメントを同期するのに役立ちます。

- 🌍 **国際化（i18n）**：サイトを複数の言語に翻訳します。

Docusaurus v2+は、すべてのユーザーにアクセシビリティが高く、稲妻のように速く動作するように生まれました。

- ⚡️ **稲妻のように速い**。Docusaurus v2+は、コンテンツを光のように速くロードする[PRPLパターン](https://developers.google.com/web/fundamentals/performance/prpl-pattern/)に従います。

- 🦖 **アクセシビリティ**。アクセシビリティに注意を払い、すべてのユーザーが平等にサイトにアクセスできるようにします。

## デザイン原則 {#design-principles}

- **学ぶことが少ない**。DocusaurusはAPIが非常に小さいため、学習して使用するのが簡単でなければなりません。ユーザーは、より多くのコードと時間が必要であっても、ほとんどのことを達成できるでしょう。間違った抽象化を使用するよりも、むしろ抽象化を使用しないほうが良いです。私たちは、ユーザーが間違った抽象化を回避するために苦労する状況を望みません。[小さなAPI範囲](https://www.youtube.com/watch?v=4anAwXYqLG8)に関する必須の講演を聴いてみてください。

- **直感的**。ユーザーは、Docusaurusプロジェクトのディレクトリを見たり、新しい機能を追加したりする際に圧倒感を感じないでしょう。直感的で馴染みのあるアプローチを使用して簡単に構築できるようにします。

- **レイヤー化されたアーキテクチャを提供**。スタックの各レイヤー（コンテンツ/テーマ/スタイリング）間の関心事の分離が明確でなければなりません。つまり、よく抽象化されモジュール化されている必要があります。

- **合理的なデフォルトを提供**。一般的で人気のあるパフォーマンス最適化と設定がユーザーのために実行されますが、ユーザーはこれらを上書きするオプションを持ちます。

- **特定のライブラリ/フレームワークに依存しない**。ユーザーは基本プラグインやCSSを使用する必要はありませんが、使用が強く推奨されます。React LoadableやReact Routerなどの特定のコアインフラは基本的なパフォーマンス最適化を実行するため置き換えることはできませんが、より高いレベルのものはそうではありません。Markdownエンジン、CSSフレームワーク、CSS方法論、その他のアーキテクチャの選択は完全にユーザー次第です。

私たちは開発者として、ライブラリがどのように動作するかを知ることが、それをより良く使用するのに役立つと信じています。したがって、私たちはDocusaurusのアーキテクチャと様々なコンポーネントを説明することに努力を注いでいます。これにより、ユーザーがツールについてより深く理解し、より熟練して使用できることを願っています。

## 他のツールとの比較 {#comparison-with-other-tools}

すべての静的サイトジェネレーターの中で、Docusaurusはドキュメントサイトに特に焦点を当てており、多くの基本機能を提供しています。

私たちは他の主要な静的サイトジェネレーターも研究し、比較に関する私たちの洞察を共有したいと思います。これにより、皆さんが様々な選択肢の中から道を見つけるのに役立つことを願っています。

### Gatsby {#gatsby}

[Gatsby](https://www.gatsbyjs.com/)は多くの機能を備えており、豊富なプラグインエコシステムを持ち、Docusaurusができることすべてを行うことができます。当然ながら、それだけ学習曲線が高くなります。Gatsbyは多くのことをうまく実行し、様々な種類のウェブサイトを構築するのに適しています。一方、Docusaurusは一つのことを非常にうまく行うことに集中しています。つまり、Docusaurusの目的は、コンテンツを作成して公開するための最高のツールになることです。

GraphQLはGatsbyの中核ですが、Gatsbyサイトを構築するために必ずしもGraphQLが必要というわけではありません。静的ウェブサイトを構築する際、ほとんどの場合、GraphQLが提供する柔軟性は必要ありません。

Docusaurus v2+の多くの側面はGatsbyの長所からインスピレーションを受けており、良い代替案となりうます。

[Docz](https://github.com/pedronauck/docz)はドキュメントウェブサイトを作成するためのGatsbyテーマです。現在、Docusaurusよりも機能が少なくなっています。

### Next.js {#nextjs}

[Next.js](https://nextjs.org/)は、もう一つの非常に人気のあるハイブリッドReactフレームワークです。良いドキュメントウェブサイトを構築するのに役立ちますが、ドキュメントのユースケースに特化しておらず、Docusaurusがデフォルトで提供するものを実装するにははるかに多くの作業が必要になるでしょう。

[Nextra](https://github.com/shuding/nextra)はNext.js上に構築された独断的な静的サイトジェネレーターです。現在、Docusaurusよりも機能が少なくなっています。

### VitePress {#vitepress}

[VitePress](https://vitepress.dev/)はDocusaurusと多くの類似点があります。どちらもコンテンツ中心のウェブサイトに大きく焦点を当てており、カスタマイズされたドキュメント機能をデフォルトで提供します。しかし、VitePressはVueをベースにしており、DocusaurusはReactをベースにしています。Vueベースのソリューションを望むなら、VitePressは良い選択となるでしょう。

### MkDocs {#mkdocs}

[MkDocs](https://www.mkdocs.org/)は、Docusaurusと同様の価値提案を持つ人気のあるPython静的サイトジェネレーターです。

シングルページアプリケーションが必要なく、Reactを活用する予定がない場合は、良い選択となるかもしれません。

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/)は美しいテーマです。

### Docsify {#docsify}

[Docsify](https://docsify.js.org/)はドキュメントウェブサイトを簡単に作成できるようにしますが、静的サイトジェネレーターではなく、SEOにフレンドリーではありません。

### GitBook {#gitbook}

[GitBook](https://www.gitbook.com/)は非常にクリーンなデザインを持っており、多くのオープンソースプロジェクトで使用されてきました。オープンソースツールよりも商業製品に焦点を移すにつれて、オープンソースプロジェクトのドキュメントサイトの要件にもはや合わなくなりました。その結果、多くのプロジェクトが他の製品に移行しました。ReduxがDocusaurusに移行した理由について[こちら](https://github.com/reduxjs/redux/issues/3161)で読むことができます。

現在、GitBookはオープンソースおよび非営利チームに対してのみ無料です。Docusaurusはすべての人に無料です。

### Jekyll {#jekyll}

[Jekyll](https://github.com/jekyll/jekyll)は最も成熟した静的サイトジェネレーターの1つであり、使用するのに素晴らしいツールでした。実際、Docusaurus以前、FacebookのほとんどのオープンソースウェブサイトはJekyllで構築されていました! 始めるのがとても簡単です。私たちはJekyllで静的サイトを構築するのと同様の開発者体験を提供したいと考えています。

静的に生成されたHTMLと`<script />`タグを使用して追加されたインタラクティビティと比較すると、Docusaurusサイトは一つのReactアプリケーションです。現代的なJavaScriptエコシステムツールを使用して、ドキュメントサイトのパフォーマンス、アセットビルドパイプライン、最適化、そして設定の容易さに関する新しい標準を設定したいと考えています。

## 何か足りないですか？ {#something-missing}

ドキュメントに問題がある場合や、ドキュメントまたはプロジェクト全般を改善できる提案がある場合は、[イシューを提起](https://github.com/facebook/docusaurus)するか、[@docusaurus](https://twitter.com/docusaurus) Twitterアカウントにメンションしてツイートを送ってください。

新機能のリクエストについては、[機能リクエストボード（Canny）](/feature-requests)に投稿することができます。これはロードマップ作成に役立つツールであり、GitHubイシューよりも整理しやすいアップボートを通じて、コアチームにどの機能が高い需要を示しているかについてより良い指標を提供します。新機能（特に大規模な機能）に関するプルリクエストは控えめにお願いします。すでに誰かが作業中かもしれませんし、私たちのロードマップの一部かもしれません。まず私たちと相談してください!
